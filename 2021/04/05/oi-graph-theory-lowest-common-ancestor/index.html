【OI考古】图论 | 最近公共祖先 LCA

发表于2021-04-05|更新于2021-07-22|编程竞赛 (Competitive Programming)图论
|字数总计:2.3k|阅读时长:10分钟|阅读量:

最近公共祖先简称 LCA（Lowest Common Ancestor）。两个节点的最近公共祖先，就是这两个点的公共祖先里面，离根最远的那个。

以人的肉眼看来，一棵树上任意两点的LCA位置是显然的，然而对于计算机来说，最朴素的算法便是遍历整棵树来寻找LCA，当询问次数很多的时候，朴素算法的效率将非常低下。本文将介绍几种常见的快速求解LCA的算法。

模板题：洛谷 P3379 | [模板] 最近公共祖先（LCA）

题目描述

如题，给定一棵有根多叉树，请求出指定两个点直接最近的公共祖先。

输入格式

第一行包含三个正整数 N,M,SN,M,SN,M,S ，分别表示树的结点个数、询问的个数和树根结点的序号。

接下来 N−1N-1N−1 行每行包含两个正整数 x,yx, yx,y ，表示 xxx 结点和 yyy 结点之间有一条直接连接的边（数据保证可以构成树）。

接下来 MMM 行每行包含两个正整数 a,ba, ba,b ，表示询问 aaa 结点和 bbb 结点的最近公共祖先。

输出格式

输出包含 MMM 行，每行包含一个正整数，依次为每一个询问的结果。

说明/提示

对于 70%70\%70% 的数据， N≤10000N\leq 10000N≤10000 ， M≤10000M\leq 10000M≤10000 。

对于 100%100\%100% 的数据， N≤500000N\leq 500000N≤500000 ， M≤500000M\leq 500000M≤500000 。

树上倍增

思路：

  1. DFSDFSDFS 计算每个节点的深度，保存在depth[i]数组中。此过程时间复杂度 O(n)O(n)O(n) 。

  2. 计算fa[i][j]数组，其表示第 iii 个节点的第 2j2^j2j 个父亲的编号，满足以下性质：

    fa[i][j]=fa[fa[i][j−1]][j−1]fa[i][j] = fa[fa[i][j-1]][j-1] fa[i][j]=fa[fa[i][j−1]][j−1]

    利用此性质可以在 DFSDFSDFS 过程中初始化 fa[i][0]fa[i][0]fa[i][0] 的值，然后逐一计算所有的 fa[i][j]fa[i][j]fa[i][j] 。此过程时间复杂度 O(nlog⁡n)O(n\log_{}{n})O(nlog​n) 。

  3. 查询 a，ba，ba，b 的 LCALCALCA 过程中，应首先用倍增法，让深度较深的节点跳至与另一节点同一高度，然后二者同事倍增，最后跳到 LCALCALCA 的子节点，此时 aaa 或 bbb 的父节点就是 LCALCALCA 。此过程时间复杂度 O(nlog⁡n)O(n\log_{}{n})O(nlog​n) 。

解决方案

Tarjan 算法

思路：

利用Tarjan算法求LCA需要结合并查集。

  1. 初始化并查集，对于每个节点 uuu ，设 fa[u]=ufa[u]=ufa[u]=u 。
  2. 在Tarjan算法的执行过程中，如果当前DFS到点 uuu ，先继续DFS其子节点 vvv ，再将 fa[v]fa[v]fa[v] 设为 uuu 。
  3. 对于以 uuu 为根的子树，考察其下的一对LCA查询 a，ba，ba，b 。若 aaa 已经被DFS完成，则记录其完成状态，当DFS至 bbb 时，发现 aaa 已被访问，此时使用并查集查询 aaa 所属并查集的根节点，由于此时以 uuu 为根的子树还没有DFS完成，即 fa[u]=ufa[u]=ufa[u]=u ，故查询结果为 uuu 本身， uuu 即为 a，ba，ba，b 的最近公共祖先LCA。

解决方案

数链剖分

思路

先对给出的树进行树链剖分，划分出重轻链，可依托以下几个数组进行划分：

  • size[]： size[u]size[u]size[u] 表示以 uuu 为根的子树的重量，每个节点对子树的贡献为 111 。
  • son[]： son[u]son[u]son[u] 为 uuu 的所有子节点中 viv_ivi​ ， size[vi]size[v_i]size[vi​] 值最大的 viv_ivi​ ， u−>viu -> v_iu−>vi​ 称为重链，一条树链由若干连续重链构成。
  • top[]： top[u]top[u]top[u] 表示 uuu 所在树链的顶部节点编号。

以上数组可以用两次DFS求得。

然后开始求LCA，对于每对LCA的询问 a，ba，ba，b ：

  • 如果 a，ba，ba，b 在同一条树链身上，则LCA为深度较浅的节点。
  • 如果 a，ba，ba，b 不在同一条树链身上，则每次让 toptoptop 的深度较小者跳至其 toptoptop 的父节点，直到两点到达同一条重链，再根据上一条求LCA即可。
